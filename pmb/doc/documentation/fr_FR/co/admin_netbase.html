Nettoyage de base

Suppression de données

Le nettoyage de base permet de nettoyer le catalogue par suppression des données redondantes ou inutilisées :

  • les auteurs inutilisés

  • les éditeurs inutilisés

  • les collections inutilisées

  • les sous-collections inutilisées

  • les catégories inutilisées

  • les titres de séries inutilisés

  • les notices sans exemplaire

  • les mots clés des notices

Il est impératif de juger toutes les répercussions de ce traitement sur votre catalogue, notamment pour les catégories inutilisées, les notices sans exemplaires et les mots-clés des notices.

Ré-indexation

Avec le nettoyage de base, vous pouvez également ré-indexer votre catalogue. Cette opération permet de recréer les index de recherche propres à PMB et peut concerner :

  • l'index global

  • tous les champs de recherche

  • les acquisitions

  • les documents numériques

Génération de données

Le nettoyage de base permet également de générer :

  • les signatures de notices (permettant un contrôle de doublons lors de la création de notices).

  • les liens entre les catégories nécessaires à l'auto-postage

  • l'année de publication des articles de périodique

  • les dates de parution des notices

La génération de l'année de publication et des dates de parution, vous permet d'effectuer des recherches sur ces informations.

Pour plus d'informations sur le dé-doublonnage des notices, voir le chapitre :
  • Dédoublonnage de notices
Pour plus d'informations sur l'auto-postage, voir le chapitre :
  • Paramétrer les thésaurus

Après un nettoyage de la base, réalisez une optimisation de la base de données ( Administration Outils Maintenance MySQL Optimiser les tables ) qui permet de reconstruire les index de la base de données.
