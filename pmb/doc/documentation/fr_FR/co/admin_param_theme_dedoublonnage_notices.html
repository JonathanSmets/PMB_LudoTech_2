Dédoublonnage de notices

Le dédoublonnage en saisie de notice permet d'éviter la création de doublons lors de la saisie.

Largement configurable, ce dédoublonnage est basé sur le paramètre notice_controle_doublons (Administration > Outils > Paramètres, paramètres généraux).

Voici une liste des principaux critères sur lesquels vous pouvez faire un contrôle pour dé-doublonnage :

  • typdoc (type de document de la notice : texte imprimé, document sonore non-musical...)

  • tit1 (titre propre)

  • tit2 (titre parallèle)

  • tit3 (titre propre d'un auteur différent)

  • tit4 (complément du titre)

  • f_tparent_id (titre de série)

  • tnvol (numéro dans la série)

  • aut0_id (auteur principal)

  • ed1_id (éditeur)

  • coll_id (collection)

  • year (année de publication)

  • lien (lien hypertexte de la notice)

Pour consulter tous les critères de dédoublonnage des notices, ouvrez dans un éditeur de texte le fichier pmb/includes/notice/notice.xml.

Voici un exemple de dédoublonnage de notices sur titre propre, auteur et année d'édition : dans Administration > Outils > Paramètres, développez paramètres généraux, dans notice_controle_doublons, saisissez "1,tit1,aut0_id,year".

Attention, ce contrôle est basé sur les signatures des notices, ces signatures contiennent les informations de contrôle de doublon. Après toute modification de ce paramètre, il faut passer par Administration > Outils > Nettoyage de base, cocher Générer toutes les signatures de notices et cliquer sur Suivant.
