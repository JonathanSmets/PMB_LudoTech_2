Exemplaires UNIMARC

L'import d'exemplaires UNIMARC[1] permet d'importer, outre les notices, les exemplaires associés à celles-ci.

Cette fonction correspond typiquement à un transfert d'ouvrages, d'une bibliothèque à une autre (BDP vers médiathèque communale) ou d'un SIGB à un autre.

Zoom

Comme pour l'import de notices, Le fichier à importer peut être transmis soit :

  • par téléchargement depuis l'interface du logiciel en cliquant sur le lien Commencer : télécharger votre fichier.

  • en déposant le fichier sur le serveur dans le répertoire pmb/admin/import (avec un client FTP par exemple), puis en cliquant sur le lien préchargement.

Les options suivantes sont possibles lors de l'import :

  • ISBN obligatoire ? :

    Permet de définir si l'ISBN doit être toujours présent dans les notices importées (ce n'est pas toujours le cas, attention aux CD audio, cédéroms et DVD ainsi qu'à la littérature grise ou non publiée).

  • Dédoublonnage sur ISBN :

    Si oui, permet de dédoublonner les notices sur l'ISBN. Les notices avec un ISBN déjà existant dans le catalogue seront ignorées. Si plusieurs notices ont le même ISBN dans le fichier, seule la première sera importée.

    Ignorer les ISSN :

    Permet d'ignorer les notices ayant pour ISBN un ISSN. Cela est notamment le cas lorsque des bulletins de périodiques sont catalogués en tant que monographies.

  • Statut de notices importées :

    Ce statut permet de d'identifier rapidement les notices importées et d'en gérer la visibilité dans l'OPAC.

  • Générer les liens entre notices ?  :

    Si oui, crée les relations entre notices lors de l'import. Ce qui permet par exemple de générer le périodique et le bulletin desquels est issu un article.

  • Choisissez votre propriétaire et un statut  :

    Le propriétaire peut par exemple être la BDP[2] de votre département si vous importez des exemplaires prêtés par celle-ci, ou encore le nom d'un organisme déposant qui vous permet de mettre un fonds à disposition de vos lecteurs, etc....

    Pour le statut il est recommandé de choisir un statut temporaire. Vous changerez ensuite le statut des exemplaires lors du pointage, ce qui permet de vérifier que les exemplaires annoncés sont bien présents physiquement. Voir plus loin la section Pointage des importés.

  • Choisissez une localisation pour les exemplaires  :

    Si vous gérez différentes localisations, ceci permet dès l'import de choisir la localisation des exemplaires qui sont importés.

  • Cote obligatoire ? : 

    Dans PMB, chaque exemplaire doit comporter au moins un code-barres et une cote. Toutefois l'import permet d'insérer des exemplaires dont la cote n'a pas été renseignée.

  • Types de document (supports) / Codes statistiques / Sections :

    A chacune de ces informations est associé un codage, ce codage pouvant être générique, ou lié à un propriétaire (spécifique).

    En fonction du choix effectué, les codages rencontrés dans le fichier permettront de faire le lien avec les Supports, Codes statistiques et Sections correspondant définis dans le logiciel.

Pour chacune de ces données, PMB va examiner les notices importées. Selon le codage choisi pour les documents importés, une correspondance va être faite avec la valeur contenue dans la zone 995 du fichier UNIMARC[1] et les codages définis dans PMB.

Ainsi, pour importer un exemplaire de la BDP[2] dont le code statistique est "a", la correspondance va être faite avec la table des codes statistiques des exemplaires ( Administration  Exemplaires Codes statistiques). Si dans votre table, il existe un codage "a" dont le propriétaire est BDP et dont le libellé est "Adultes BDP", alors votre exemplaire importé aura pour code statistique "Adultes BDP".

Sélectionnez le fichier à importer en cliquant sur Parcourir. (Ce bouton peut porter un autre nom selon le navigateur utilisé), puis sur Suivant pour lancer l'import.

L'import des exemplaires commence par le transfert du fichier sur le serveur. Vous pouvez suivre le déroulement du transfert grâce à l'affichage du nombre de caractères du fichier restant à

traiter.

Une fois le transfert du fichier réalisé, l'import commence. Vous pouvez suivre le déroulement de l'import grâce à l'affichage du nombre de notices restant à traiter.

A l'issue de l'import des exemplaires un écran affiche les résultats de l'import.

Références :
  1. UNIMARC

    Basé sur le format MARC (MAchine-Readable Cataloging), UNIMARC en est une variante créé par l'IFLA en 1977. Ce format est le format officiel d'échange de données bibliographiques en France.

  2. BDP

    Bibliothèque Départementale de Prêt
